# Onboarding Playbook

Welcome to the Hans Restaurant App team! This playbook will guide you through your first days and weeks.

## Week 1: Environment & Access

### Day 1-2: Access & Setup
- [ ] GitHub access and 2FA setup
- [ ] Linear account and project access
- [ ] Slack workspace invitation
- [ ] Doppler access for secrets
- [ ] Supabase project access
- [ ] Vercel team access
- [ ] Sentry project access
- [ ] PostHog access

### Day 3-5: Development Environment
- [ ] Clone repository and run setup
- [ ] Complete [Getting Started](/getting-started) guide
- [ ] Set up local development environment
- [ ] Run all tests and verify everything works
- [ ] Create your first PR (documentation update)

## Week 2: Code & Architecture

### Understanding the Codebase
- [ ] Review [Architecture](/architecture) documentation
- [ ] Read through existing code structure
- [ ] Understand the monorepo setup
- [ ] Learn about our [Project Tooling](/tooling)

### First Tasks
- [ ] Pick up a "good first issue" from Linear
- [ ] Work on a small feature or bug fix
- [ ] Go through code review process
- [ ] Deploy your first feature

## Week 3-4: Deep Dive

### Domain Knowledge
- [ ] Learn about restaurant operations
- [ ] Understand our business requirements
- [ ] Review existing ADRs in `/adr` folder
- [ ] Shadow a senior developer

### Advanced Setup
- [ ] Set up monitoring and alerting
- [ ] Learn about our deployment process
- [ ] Understand our testing strategy
- [ ] Review security practices

## Mentorship

You'll be paired with a senior developer who will:
- Help with technical questions
- Review your first few PRs
- Guide you through the codebase
- Provide feedback and support

## Questions?

Don't hesitate to ask questions! We prefer you ask early and often rather than getting stuck.

- Use Slack for quick questions
- Create Linear issues for longer discussions
- Schedule 1:1s with your mentor
